# Group Chat Tournament Bot / 群聊锦标赛机器人

> 在微信群/Telegram 群中发起预测锦标赛，自动统计和排名

## 解决什么问题

加密社群（微信群、Telegram 群）经常自发组织预测比赛，但缺乏自动化工具来统计结果和排名。Group Chat Tournament Bot 让群管理员可以一键发起预测锦标赛，群成员在群内直接下注参与，Bot 自动追踪所有参与者的预测结果，实时更新排行榜，锦标赛结束后自动结算并公布排名。

## 核心功能

- 一键发起锦标赛：管理员设置周期（日/周/月）和规则
- 群内下注：成员发送简单格式消息参与预测
- 自动结算：到期后 Bot 自动验证结果并更新排名
- 奖池管理：支持自定义奖池分配规则（赢家通吃/按比例）

## 使用的 DeepBook / Sui 能力

| 能力 | 用途 |
|------|------|
| predict::mint | 执行锦标赛参与者的预测交易 |
| predict::redeem_permissionless | 到期后赎回结算 |
| PredictManager | 管理锦标赛资金和仓位 |
| OracleSVI | 验证预测结果是否正确 |

## 实现方案

### Bot / Keeper / 服务端
- 多平台适配器：Telegram Bot API + 微信 Webhook
- 锦标赛管理器：创建/启动/结束锦标赛的生命周期管理
- 下注解析器：解析群内消息格式的预测指令
- 结果验证器：对比预言机价格验证预测结果
- 排名引擎：实时计算参与者积分和排名
- 奖池分配器：根据规则分配奖池

### 前端 / 后端
- 技术栈：Node.js + PostgreSQL + Redis
- 群内 Bot 界面：指令、结果卡片、排行榜
- Web 仪表盘：锦标赛详情、参与者统计、历史记录
- 管理面板：创建锦标赛、配置规则、管理奖池

## 技术架构

管理员发起锦标赛 → 配置规则和奖池 → 群成员发送预测指令 → 下注解析器提取参数 → predict::mint 执行交易 → 到期时结果验证器对比预言机 → 更新排名 → 奖池分配器结算 → 群内推送最终排名。
