# Streaks Leaderboard PWA / 连胜排行榜 PWA

> 每日二元选择，连胜记录，每周奖池，NFT 徽章

## 解决什么问题

预测市场需要游戏化机制来吸引和留住普通用户。Streaks Leaderboard PWA 将 Predict 交易转化为简单的每日二元选择（"BTC 收盘涨还是跌"），通过连胜记录、每周奖池和 NFT 徽章构建留存循环。移动优先的 PWA 设计让用户可以像刷社交应用一样参与预测市场，降低 Web3 入门门槛。

## 核心功能

- 每日二元选择：基于 Predict mint/redeem 流程的简化预测界面
- 连胜系统：追踪每个用户的连续正确预测次数
- 每周奖池：连胜用户按排名分配奖励池
- NFT 徽章：链上铸造连胜里程碑徽章（5 连胜、10 连胜等）

## 使用的 DeepBook / Sui 能力

| 能力 | 用途 |
|------|------|
| predict::mint | 用户选择后执行预测交易 |
| predict::redeem_permissionless | 结算后自动赎回 |
| PredictManager | 管理用户仓位 |
| Sui NFT | 铸造连胜里程碑徽章 |

## 实现方案

### 智能合约 (Move)
- `streak_badge` 模块：连胜 NFT 徽章铸造和验证
- `weekly_pool` 模块：每周奖池管理和分配
- 关键数据结构：`StreakRecord`（用户、连胜数、最高记录）、`Badge`（NFT 元数据）

### 前端 / 后端
- 技术栈：React PWA + Workbox + Node.js
- 每日预测界面：简单的涨/跌选择卡片
- 连胜排行榜：按当前连胜和历史连胜排名
- 徽章收集页：展示用户所有链上徽章
- 每周总结：本周胜率、排名变化、奖励分配
- 离线缓存：Service Worker 支持离线查看历史

## 技术架构

用户打开 PWA → 每日预测界面展示当前 BTC 行情 → 用户选择涨/跌 → 后端调用 predict::mint 执行交易 → 到期时自动 redeem → 更新连胜记录 → 达到里程碑时铸造 NFT 徽章 → 排行榜实时更新。
