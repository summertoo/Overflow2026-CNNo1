# Mobile Trading UI / 移动优先交易界面

> 针对手机优化的 Predict 交易 UI，滑动手势下单

## 解决什么问题

现有 DeFi 界面大多针对桌面设计，在手机上操作体验差——按钮小、信息密集、交互复杂。移动优先交易界面专为手机屏幕设计，利用滑动手势（左滑看跌、右滑看涨）简化交易操作，配合大字体、卡片式布局和触控优化，让 Predict 交易像刷短视频一样轻松自然。

## 核心功能

- 滑动手势下单：右滑看涨、左滑看跌，滑动距离控制金额
- 卡片式行情：每个到期日一张卡片，上下滑动切换
- 触控优化：大按钮、拇指区域操作、触觉反馈
- 钱包集成：Sui Wallet 移动端深度链接

## 使用的 DeepBook / Sui 能力

| 能力 | 用途 |
|------|------|
| predict::mint | 手势下单后执行交易 |
| predict::redeem_permissionless | 一键赎回到期仓位 |
| PredictManager | 管理用户仓位和资金 |
| OracleSVI | 展示实时行情数据 |

## 实现方案

### 前端 / 后端
- 技术栈：React Native / Capacitor + Framer Motion
- 手势引擎：基于 Hammer.js / react-native-gesture-handler
- 卡片 UI：Tinder 风格的滑动卡片，每张代表一个预测选项
- 行情渲染：简化版 K 线 + 关键数据大字展示
- 钱包连接：Sui Wallet / Ethos 深度链接
- 推送通知：仓位到期和结算结果推送

### Bot / Keeper / 服务端
- 行情聚合器：从 predict-server 获取实时数据
- 交易代理：处理手势意图并构建 PTB 提交
- 仓位管理 API：查询用户活跃仓位和历史

## 技术架构

用户打开 App → 行情聚合器加载 BTC 价格和到期日 → 卡片 UI 展示预测选项 → 用户滑动手势选择方向和金额 → 交易代理构建 PTB → 调用 predict::mint 执行 → 结算推送通知。
