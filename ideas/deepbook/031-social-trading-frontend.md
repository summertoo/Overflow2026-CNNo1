# Social Trading Frontend / 社交交易前端

> 关注顶级预测者，一键跟单其仓位，查看其历史胜率

## 解决什么问题

加密市场中大多数散户缺乏专业分析能力，但信息极度不对称——顶级交易者的策略往往远优于普通用户。Social Trading Frontend 将 Predict 交易与社交网络结合，用户可以关注链上顶级预测者，查看其实时仓位和历史胜率，一键跟单复制其仓位。通过透明化交易者的链上记录，建立可信的社交交易生态。

## 核心功能

- 交易者排行榜：按胜率、收益率、连续正确次数排名
- 一键跟单：自动复制选定交易者的仓位比例和参数
- 实时仓位展示：查看关注交易者的当前活跃仓位
- 历史记录分析：每个交易者的详细历史胜率和盈亏曲线

## 使用的 DeepBook / Sui 能力

| 能力 | 用途 |
|------|------|
| predict::mint | 跟单时执行交易 |
| predict::redeem_permissionless | 结算后赎回 |
| PredictManager | 追踪交易者仓位 |
| 链上事件 | 分析交易者历史记录 |

## 实现方案

### 智能合约 (Move)
- `copy_trade` 模块：跟单配置和执行逻辑
- `trader_registry` 模块：交易者注册和验证
- 关键数据结构：`FollowConfig`（跟单比例、最大金额、止损）、`TraderScore`（胜率、收益排名）

### Bot / Keeper / 服务端
- 仓位追踪器：实时监控被关注交易者的 mint 事件
- 跟单执行器：检测到新仓位后按配置自动跟单
- 分析引擎：计算每个地址的历史胜率和风险指标
- 通知服务：跟单交易者开仓/平仓时推送通知

### 前端 / 后端
- 技术栈：React + ECharts + Node.js + PostgreSQL
- 交易者排行榜和详情页
- 实时仓位跟踪面板
- 跟单配置界面（金额、比例、止损）
- 个人交易记录和盈亏分析

## 技术架构

索引器追踪链上 mint 事件 → 分析引擎计算交易者排名 → 仓位追踪器监控被关注者 → 检测到新仓位 → 跟单执行器调用 predict::mint → 前端展示实时跟单状态和交易者分析。
