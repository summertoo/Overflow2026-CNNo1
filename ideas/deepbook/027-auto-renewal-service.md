# Auto-Renewal Service / 自动续期服务

> 监控即将到期的仓位，自动按策略续期或平仓

## 解决什么问题

Predict 的区间和仓位有固定到期日，到期后需要手动续期或平仓，错过时机可能导致资金闲置或策略中断。特别是运行持续策略（如做市、区间金库）的用户，需要自动化续期来维持仓位。Auto-Renewal Service 监控所有即将到期的仓位，根据预设策略自动续期到下一个周期或在特定条件下平仓。

## 核心功能

- 监控用户所有活跃仓位的到期时间，提前通知或自动处理
- 支持自定义续期策略：固定行权价、跟随 ATM、SVI 1σ 调整
- 到期前自动 mint 新区间并赎回到期仓位（原子 PTB 操作）
- 策略暂停条件：市场剧烈波动时暂停续期，等待稳定后恢复

## 使用的 DeepBook / Sui 能力

| 能力 | 用途 |
|------|------|
| predict::mint | 自动续期时 mint 新区间 |
| predict::redeem_permissionless | 赎回到期仓位释放资金 |
| PTB | 赎回 + mint 打包为原子操作 |
| OracleSVI | 计算 ATM 行权价和续期参数 |

## 实现方案

### 智能合约 (Move)
- `renewal_config` 模块：用户注册续期策略和参数
- `strategy_registry` 模块：预设策略模板（固定、ATM 跟踪、动态）
- 关键数据结构：`RenewalPlan`（仓位 ID、策略类型、参数）、`RenewalHistory`（续期记录）

### Bot / Keeper / 服务端
- 到期监控器：持续扫描即将到期的仓位
- 策略执行器：根据用户配置计算新区间参数
- 续期构建器：组装 redeem（旧仓位）+ mint（新区间）的 PTB
- 条件检查器：评估市场条件是否满足续期要求
- 通知服务：续期成功/失败时通知用户

### 前端 / 后端
- 技术栈：React + Node.js + PostgreSQL
- 策略配置界面：选择续期策略、设置参数
- 仓位时间线：显示所有仓位到期时间和续期计划
- 续期历史记录和成功率统计

## 技术架构

到期监控器持续扫描仓位 → 发现即将到期的仓位 → 查找用户续期配置 → 策略执行器计算新区间参数 → 条件检查器评估市场 → 续期构建器组装赎回+mint PTB → 提交链上执行 → 通知用户结果。
