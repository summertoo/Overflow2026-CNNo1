# DeFi 策略市场 / DeFi Strategy Marketplace

> 策略创作者发布收益策略，用户一键跟投，创作者赚取管理费

## 解决什么问题

DeFi 用户中只有少数专家有能力设计出优秀的收益策略，大多数人不知道如何优化资产配置。但这些专家的策略缺乏标准化的分享和变现渠道。DeFi 策略市场让策略创作者将自己的收益策略编码并发布到链上，其他用户可以一键跟投，策略自动执行，创作者从中收取管理费。这为专家创造了收入来源，也为普通用户提供了专业级的投资策略。

## 核心功能

- 策略发布：创作者定义策略逻辑、风险等级、管理费率和最低跟投金额
- 一键跟投：用户浏览策略市场，选择策略后存入资金自动跟投
- 自动执行：策略在链上自动执行，跟投者无需手动操作
- 业绩排行榜：按收益率、Sharpe 比率等指标排名，帮助用户选择优质策略
- 策略回测工具：创作者可在发布前用历史数据回测策略表现，验证有效性
- 资金安全隔离：每个策略的资金独立管理，策略间互不影响

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Move 对象模型 | Strategy 作为独立 Object，存储策略逻辑和参数 |
| PTB | 原子化执行"跟投资金分配 → 策略执行 → 管理费扣除" |
| Coin 标准 | 统一处理策略中的多种资产 |
| 动态字段 | 存储跟投者列表、管理费累计和业绩数据 |
| 共享对象 | 策略市场作为共享对象，支持浏览和跟投 |

## 实现方案

### 智能合约 (Move)
- `strategy_factory` 模块：创作者发布新策略，定义执行逻辑和参数
- `follower_manager` 模块：管理跟投者注册、资金分配和退出
- `fee_collector` 模块：计算和扣除管理费（基于 AUM 或收益分成）
- `performance_tracker` 模块：计算策略净值和收益率
- 关键数据结构：`Strategy { creator: address, logic: StrategyLogic, risk_level: u8, management_fee: u64, min_investment: u64, total_aum: Coin<T>, followers: u32 }`

### 前端 / 后端
- 技术栈：Next.js + @mysten/dapp-kit + Recharts
- 关键页面：Strategy Market（策略市场）、Strategy Detail（策略详情和业绩）、My Strategies（我的策略/跟投）、Creator Dashboard（创作者管理后台）
- 后端 Keeper 执行策略操作并计算业绩指标
- 支持策略创作者设置锁仓期和最低跟投金额，灵活管理资金规模

## 技术架构

策略创作者通过合约发布策略：定义执行逻辑（如"每周在 Scallop 和 Cetus 间轮动"）、风险等级、管理费率。用户浏览策略市场，查看历史业绩和风险指标后选择跟投，通过 PTB 将资金存入策略资金池。Keeper 按策略逻辑定期执行操作，所有跟投者的资金按比例同步调整。管理费自动从跟投者资金中扣除并转入创作者地址。前端展示实时策略净值和收益率排行。
