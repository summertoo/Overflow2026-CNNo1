# 链上基金管理平台 / On-Chain Fund Management Platform

> 创建、管理、审计链上投资基金，LP 份额代币化可流通

## 解决什么问题

传统投资基金的运营依赖多个中介（基金管理人、托管人、审计机构），运营成本高且不够透明。投资者难以实时了解基金持仓和业绩，赎回流程复杂且周期长。链上基金管理平台将基金的创建、募集、投资和赎回全流程搬到链上，LP 份额代币化后可在 DEX 自由交易，持仓和业绩完全透明可审计。

## 核心功能

- 基金创建与募集：基金经理创建基金，设置策略和费率，向公众募集 LP 资金
- 链上投资执行：基金经理通过 PTB 执行投资操作，所有交易在链上可追溯
- LP 份额代币：投资者获得可转让的 LP 代币，可在 DEX 上自由交易
- 链上审计：基金持仓和交易历史完全公开，支持第三方审计验证
- 费率透明化：管理费和业绩分成在链上自动计算扣除，投资者可实时查看
- 赎回队列机制：大额赎回进入队列分批执行，保护剩余投资者利益

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Move 对象模型 | FundVault 作为共享对象，管理基金资产和状态 |
| PTB | 原子化执行"申购 → 分配份额 → 投资操作 → 管理费计算" |
| Coin 标准 | LP 份额代币使用 Coin<T>，支持 DEX 交易 |
| 动态字段 | 存储持仓明细、NAV 计算和费率参数 |
| 共享对象 | 基金池允许多 LP 并发申购赎回 |

## 实现方案

### 智能合约 (Move)
- `fund_factory` 模块：创建新基金，定义策略类型和费率结构
- `subscription` 模块：处理 LP 申购和赎回，铸造/销毁份额代币
- `nav_calculator` 模块：基于链上持仓计算基金净值
- `investment_manager` 模块：执行基金经理的投资操作
- 关键数据结构：`FundVault { manager: address, strategy: String, nav_per_share: u64, total_supply: u64, holdings: vector<Position>, management_fee: u64, performance_fee: u64 }`

### 前端 / 后端
- 技术栈：Next.js + @mysten/dapp-kit + Recharts + PDF 导出
- 关键页面：Fund Explorer（基金浏览）、Fund Detail（基金详情和持仓）、Subscribe/Redeem（申购赎回）、Manager Dashboard（基金经理管理台）、Audit Report（审计报告）
- 后端计算 NAV 和生成审计报告
- 支持基金管理人设置申购开放期和最低认购金额，灵活控制基金规模

## 技术架构

基金经理通过合约创建基金，设定策略类型（多空、量化、套利）、管理费和业绩分成。投资者浏览基金市场后申购：存入资金 → 合约铸造等比例 LP 代币。基金经理通过 PTB 执行投资操作，所有交易在链上记录。NAV 由 nav_calculator 根据链上持仓和预言机价格实时计算。赎回时销毁 LP 代币，按当前 NAV 返还等值资产。LP 代币可在 DEX 上交易，为投资者提供流动性退出通道。
