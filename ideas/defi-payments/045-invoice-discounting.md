# 票据贴现平台 / Invoice Discounting Platform

> 企业应收账款代币化，投资者购买票据提供流动性，到期自动结算

## 解决什么问题

中小企业经常面临现金流问题——已经交付货物或服务但应收账款还没到期（30-90 天），急需资金周转却无法从银行获得短期贷款。票据贴现平台将企业的应收账款代币化为链上票据，投资者提前买入票据为企业提供流动性，到期时合约自动从付款方收取全额账款，投资者赚取贴现利息。

## 核心功能

- 票据创建：企业上传应收账款信息，合约代币化为链上票据
- 贴现交易：投资者以折扣价购买票据，为企业提供即时流动性
- 到期自动结算：票据到期时合约自动从付款方地址收取全额款项
- 风险评级：基于付款方链上信用历史为票据评定风险等级
- 票据拆分投资：大额票据可拆分为多个份额，降低单一投资者风险
- 逾期自动处理：付款方到期未付款自动触发逾期流程，保证金补偿投资者

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Move 对象模型 | Invoice 作为独立 Object，存储票据信息和状态 |
| PTB | 原子化执行"创建票据 → 贴现购买 → 到期结算" |
| Coin 标准 | 票据金额和贴现使用 Coin<T> |
| 时间锁 | 基于时间戳控制票据到期和结算 |
| 动态字段 | 存储付款方信息、风险评级和贴现历史 |

## 实现方案

### 智能合约 (Move)
- `invoice_factory` 模块：创建应收账款票据，记录金额、到期日和付款方
- `discount_market` 模块：票据贴现交易市场，支持竞价和多投资者分购
- `settlement_engine` 模块：到期时自动执行结算
- `risk_scoring` 模块：基于链上历史评定付款方信用和票据风险
- 关键数据结构：`Invoice { issuer: address, payer: address, amount: Coin<T>, discount_rate: u64, maturity: u64, status: u8, investors: vector<address> }`

### 前端 / 后端
- 技术栈：Next.js + @mysten/dapp-kit + Sui TypeScript SDK
- 关键页面：Create Invoice（创建票据）、Market（票据市场）、My Invoices（我的票据）、Portfolio（投资组合）、Settlement History（结算历史）
- 后端提供付款方信用评估和到期提醒服务
- 票据创建时支持上传发票文件哈希，关联链下原始凭证

## 技术架构

企业创建票据时通过 PTB 锁定保证金并记录应收账款信息（付款方地址、金额、到期日）。risk_scoring 根据付款方的链上历史计算信用评分和风险等级。投资者在市场中浏览票据，以折扣价购买（如面值 1000 USDC 的票据以 970 USDC 购买）。到期日，settlement_engine 通过 PTB 从付款方地址自动扣除全额款项并分配给投资者。如果付款方违约，保证金用于补偿投资者。
