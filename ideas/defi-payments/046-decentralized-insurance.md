# 去中心化保险协议 / Decentralized Insurance Protocol

> 用户购买保险，保费池投资生息，理赔条件触发自动赔付

## 解决什么问题

DeFi 用户面临智能合约漏洞、预言机操纵和协议攻击等风险，但缺乏有效的保险保护。现有链上保险产品覆盖范围有限、保费高、理赔流程慢且争议多。去中心化保险协议让用户为各种链上风险购买保险，保费池自动投资生息扩大赔付能力，理赔条件（预言机价格异常、合约暂停等）触发后自动赔付，无需人工审核。

## 核心功能

- 多类型保险：支持智能合约保险、稳定币脱锚保险、清算保护保险等
- 保费池投资：收取的保费自动投入低风险 DeFi 策略生息
- 自动理赔：当链上条件触发（如预言机价格异常）时自动执行赔付
- 风险分级：根据风险等级差异化定价保费
- 再保险机制：大额保单可由多个保费池共同承保，分散赔付风险
- 社区治理投票：重大理赔争议由社区质押者投票决定

## 使用的 Sui 原语

| 原语 | 用途 |
|------|------|
| Move 对象模型 | InsurancePolicy 作为独立 Object，存储保单信息 |
| PTB | 原子化执行"保费支付 → 理赔触发 → 赔付转账" |
| Coin 标准 | 保费和赔付使用 Coin<T> |
| 预言机集成 | 通过 Pyth 获取链上数据判断理赔条件 |
| 动态字段 | 存储保单条款、理赔记录和保费池状态 |

## 实现方案

### 智能合约 (Move)
- `insurance_factory` 模块：创建保险产品，定义覆盖范围和定价模型
- `policy_manager` 模块：管理保单的购买、续期和到期
- `premium_pool` 模块：管理保费资金池，投资低风险策略
- `claim_processor` 模块：验证理赔条件并自动执行赔付
- 关键数据结构：`InsurancePolicy { holder: address, coverage_type: u8, coverage_amount: Coin<T>, premium: Coin<T>, start_date: u64, end_date: u64, claim_conditions: vector<Condition> }`

### 前端 / 后端
- 技术栈：Next.js + @mysten/dapp-kit + Sui TypeScript SDK
- 关键页面：Insurance Market（保险市场）、Buy Policy（购买保险）、My Policies（我的保单）、Claim History（理赔记录）、Pool Stats（保费池状态）
- 后端监控链上条件并触发理赔
- 保费池投资策略仅限低风险协议（借贷池、稳定币收益），保障赔付能力

## 技术架构

保险产品创建时定义覆盖类型、理赔条件（如预言机价格偏离 >10%）和定价模型。用户购买保险时通过 PTB 支付保费并收到 InsurancePolicy 对象。保费收入汇集到 premium_pool 中，自动投入低风险借贷协议生息。当链上条件触发理赔时（如目标协议的预言机检测到异常），claim_processor 自动验证条件并执行赔付。保费池的投资收益用于扩大赔付能力和降低保费。
