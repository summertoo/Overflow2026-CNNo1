# Verifiable Inference Proof / 可验证推理证明

> AI 推理的输入输出存储在 Walrus，链上记录哈希，任何人可验证推理结果未被篡改

## 解决什么问题

AI 模型的推理过程是不透明的黑盒，用户无法验证模型提供者是否真的使用了声称的模型进行推理。推理结果可能在传输过程中被篡改，导致基于该结果的决策不可信。本方案将 AI 推理的完整输入、模型参数摘要和输出结果存储在 Walrus，哈希上链，建立可公开验证的推理证据链，让 AI 推理结果具备可证明的可信度。

## 核心功能

- 推理记录存证：每次 AI 推理自动记录完整输入、模型标识和输出到 Walrus
- 链上哈希验证：推理记录的哈希上链，任何人可独立验证数据完整性
- 推理审计追踪：支持查询某次决策背后的完整推理证据链

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 存储推理输入、输出、模型配置等完整记录 |
| Sui 对象模型 | 链上记录推理哈希、模型注册、验证次数 |
| Sui 事件机制 | 推理完成和验证通过时触发链上事件 |
| Seal 加密 | 加密包含商业机密的推理输入 |

## 实现方案

### Agent / AI 部分

- 推理代理 Agent 执行实际的 AI 推理任务，记录完整输入输出
- 摘要 Agent 提取模型参数哈希和关键配置，生成模型指纹
- 证据 Agent 组装推理证据包（输入哈希 + 模型指纹 + 输出哈希 + 时间戳）
- 验证 Agent 响应外部验证请求，重算哈希与链上记录比对
- 审计 Agent 定期分析推理模式，检测异常或作弊行为

### Walrus 集成

- 推理输入以 `inference/{id}/input.json.enc` 加密存储
- 推理输出以 `inference/{id}/output.json` 存储
- 模型配置以 `inference/{id}/model_config.json` 存储
- 证据包以 `inference/{id}/proof.json` 存储完整证据链
- 推理哈希以 Sui 链上对象存储，记录输入/输出哈希和模型指纹

### 前端 / 后端

- 前端：React 推理验证面板，支持提交验证请求和查看证据链
- 后端：Python 推理服务 + Move 验证合约
- 关键页面：推理提交页、验证结果页、证据链查看页、审计日志页

## 技术架构

用户发起推理请求 → 推理代理 Agent 执行推理 → 记录完整输入和输出 → 摘要 Agent 生成模型指纹 → 证据 Agent 组装证据包 → 推理记录写入 Walrus → 计算哈希提交到 Sui 链上 → 验证者请求验证 → 验证 Agent 从 Walrus 获取记录 → 重算哈希与链上比对 → 返回验证结果和完整证据链。
