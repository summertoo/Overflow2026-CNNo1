# Decentralized Identity Credential / 去中心化身份凭证

> 用户身份验证结果存储在 Walrus，通过 Seal 保护隐私，支持可选择性披露的身份凭证

## 解决什么问题

传统身份验证系统依赖中心化机构，用户数据存在单点泄露风险。跨平台身份验证需要重复提交材料，用户体验差且效率低。本方案利用 Walrus 和 Seal 构建去中心化身份凭证系统，用户完成一次验证后获得链上凭证，可选择性向第三方披露身份属性（如"已满18岁"而不暴露具体生日），实现隐私保护下的可信身份验证。

## 核心功能

- 可验证凭证颁发：验证机构完成身份验证后颁发 W3C 标准可验证凭证，存储在 Walrus
- 选择性披露：用户可向第三方证明特定属性而不暴露完整身份信息
- 凭证撤销与更新：支持凭证过期、撤销和更新，链上实时验证凭证状态

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 存储身份凭证文档、验证材料、凭证历史 |
| Sui 对象模型 | 链上记录凭证注册、DID 标识、撤销状态 |
| Seal 加密 | 加密凭证中的敏感身份信息，仅授权方可解密 |
| zkLogin | 支持 OAuth 登录方式关联去中心化身份 |

## 实现方案

### Agent / AI 部分

- 验证 Agent 自动审核用户提交的身份材料，进行 KYC/AML 检查
- 凭证签发 Agent 根据验证结果生成 W3C 可验证凭证
- 披露控制 Agent 帮助用户选择要披露的属性，生成零知识证明
- 撤销监控 Agent 持续监控凭证状态，检测异常使用或过期凭证
- 合规 Agent 确保证书颁发流程符合各地区法规要求

### Walrus 集成

- 身份凭证以 `identity/{did}/credentials/{cred_id}.json.enc` 加密存储
- 验证材料以 `identity/{did}/evidence/{batch}.json.enc` 存储
- 凭证历史以 `identity/{did}/history.json` 记录颁发和撤销事件
- DID 文档以 Sui 链上对象存储，记录公钥和验证方法
- 撤销注册表以 Sui 链上维护，实时更新凭证状态

### 前端 / 后端

- 前端：React 身份钱包界面，支持凭证管理、选择性披露、验证请求
- 后端：Node.js 凭证服务 + Move 智能合约
- 关键页面：身份钱包页、凭证列表页、披露授权页、验证结果页

## 技术架构

用户提交身份材料 → 验证 Agent 审核材料 → 合规 Agent 检查法规 → 通过后凭证签发 Agent 生成 VC → Seal 加密敏感字段 → 凭证文档写入 Walrus → Sui 链上注册 DID 和凭证哈希 → 第三方请求验证 → 披露控制 Agent 生成最小化披露 → 零知识证明提交 → 第三方通过链上验证凭证有效性和签名。
