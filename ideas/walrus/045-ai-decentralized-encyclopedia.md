# AI Decentralized Encyclopedia / AI 驱动的去中心化百科

> 条目由 AI 生成和更新，所有版本存储在 Walrus，社区审核后上链确认，构建可信知识库

## 解决什么问题

传统百科全书依赖人工编辑，内容更新慢且覆盖面有限。中心化百科平台存在审核偏见和单点控制问题。AI 生成内容的准确性难以验证，缺乏透明的审核追踪。本方案构建 AI 驱动的去中心化百科全书，AI 自动生成和更新条目，所有历史版本存储在 Walrus 确保不可篡改，社区通过链上投票审核确认内容的准确性。

## 核心功能

- AI 条目生成：Agent 自动研究主题并生成结构化百科条目，附带引用来源
- 版本追踪：每次条目修改都保存为独立版本存储在 Walrus，可追溯任意历史版本
- 社区审核投票：条目更新需经过社区审核员链上投票确认，达到阈值后才生效

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 存储百科条目内容、引用来源、审核记录 |
| Sui 对象模型 | 链上记录条目索引、版本哈希、审核状态 |
| Sui 投票机制 | 社区审核员通过链上投票确认条目准确性 |
| Walrus 不可篡改 | 确保已审核条目版本永久保存不可修改 |

## 实现方案

### Agent / AI 部分

- 研究 Agent 根据主题自动搜集多语言资料，整理关键信息
- 写作 Agent 将研究资料组织为结构化百科条目，附引用来源
- 事实核查 Agent 验证条目中的关键陈述，交叉验证来源可信度
- 更新 Agent 定期检查已发布条目是否需要更新，识别过时信息
- 翻译 Agent 将审核通过的条目翻译为多语言版本

### Walrus 集成

- 条目内容以 `wiki/{topic_id}/content.json` 存储当前版本
- 历史版本以 `wiki/{topic_id}/history/v{N}.json` 存储所有旧版本
- 引用来源以 `wiki/{topic_id}/references.json` 存储
- 审核记录以 `wiki/{topic_id}/reviews/{review_id}.json` 存储
- 条目索引以 Sui 链上对象存储，记录版本哈希和审核状态

### 前端 / 后端

- 前端：React 百科阅读器 + 编辑器，支持条目浏览、编辑、审核
- 后端：Python AI 生成服务 + Move 审核合约
- 关键页面：条目阅读页、编辑提交页、审核投票页、历史版本页

## 技术架构

用户提交主题请求 → 研究 Agent 搜集资料 → 写作 Agent 生成条目初稿 → 事实核查 Agent 验证准确性 → 条目写入 Walrus → Sui 链上记录待审核状态 → 社区审核员查看并投票 → 达到阈值后链上确认 → 更新 Agent 定期检查过时内容 → 提交更新提案 → 再次审核投票 → 新版本写入 Walrus → 链上更新版本哈希。
