# Medical Data Collaboration / 医疗数据协作平台

> 脱敏医疗数据存储在 Walrus，通过 Seal 保护隐私，AI Agent 辅助分析，推动医疗研究协作

## 解决什么问题

医疗数据高度敏感，医院和研究机构之间难以安全共享数据进行联合研究。患者对自身医疗数据缺乏控制权，不知道数据被谁使用。医疗研究中大量数据孤岛导致样本量不足，影响研究质量。本方案利用 Walrus 存储脱敏医疗数据，Seal 加密保护隐私，AI 辅助数据分析，在严格保护患者隐私的前提下推动跨机构医疗数据协作。

## 核心功能

- 隐私脱敏处理：AI 自动识别和脱敏医疗数据中的患者身份信息（PHI），确保合规
- 安全数据协作：多机构通过 Walrus 共享脱敏数据集，Seal 加密控制访问权限
- AI 辅助分析：Agent 在授权数据上执行统计分析、趋势发现和辅助诊断建议

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 存储脱敏医疗数据集、分析报告、研究结果 |
| Sui 对象模型 | 链上记录数据授权、使用许可、研究项目注册 |
| Seal 加密 | 加密医疗数据，按授权策略控制解密权限 |
| Sui 事件机制 | 数据使用请求和授权变更时触发通知 |

## 实现方案

### Agent / AI 部分

- 脱敏 Agent 自动识别医疗记录中的患者身份信息并替换为匿名标识
- 合规 Agent 检查数据处理流程是否符合 HIPAA 等医疗隐私法规
- 统计分析 Agent 在授权范围内执行统计分析和流行病学研究
- 辅助诊断 Agent 基于脱敏数据训练和验证辅助诊断模型
- 数据质量 Agent 评估共享数据集的完整性和一致性

### Walrus 集成

- 脱敏数据以 `medical/{study_id}/datasets/{version}.csv.enc` 加密存储
- 分析报告以 `medical/{study_id}/reports/{report_id}.json` 存储
- 研究协议以 `medical/{study_id}/protocol.json.enc` 加密存储
- 患者同意书记录以 Sui 链上对象存储，记录授权范围和有效期
- 数据使用日志以 `medical/{study_id}/access_log.json.enc` 加密记录

### 前端 / 后端

- 前端：React 医疗协作平台，支持数据上传、研究管理、分析可视化
- 后端：Python 医疗 AI 服务 + Move 授权合约
- 关键页面：数据提交页、研究项目管理页、分析结果页、授权管理页

## 技术架构

医院提交医疗数据 → 脱敏 Agent 识别并脱敏 PHI → 合规 Agent 验证 HIPAA 合规 → Seal 加密脱敏后数据 → 加密数据写入 Walrus → Sui 链上记录数据授权 → 研究机构请求访问 → 合规 Agent 检查授权范围 → 授权后 Seal 解密对应字段 → 统计分析 Agent 执行研究分析 → 数据质量 Agent 验证结果 → 分析报告存入 Walrus → 数据使用日志链上记录。
