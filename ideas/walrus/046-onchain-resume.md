# Onchain Resume / 链上简历系统

> 用户职业数据存储在 Walrus，AI 自动生成简历，链上验证真实性，构建可信职业档案

## 解决什么问题

传统简历容易被伪造，雇主无法高效验证候选人声称的工作经历和技能。每次换工作都要重新整理简历，不同岗位需要定制化版本。职业数据分散在 LinkedIn、GitHub、各类证书平台中，缺乏统一管理。本方案将用户的职业数据统一存储在 Walrus，AI 根据目标岗位自动生成定制化简历，雇主可通过链上验证快速确认信息真实性。

## 核心功能

- 职业数据保险箱：用户的教育、工作、项目、证书等职业数据统一加密存储在 Walrus
- AI 简历生成：根据目标岗位 JD 自动从数据保险箱中提取相关经历，生成定制化简历
- 链上可信验证：雇主和背景调查机构可通过链上凭证验证简历信息的真实性

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 存储用户职业数据、简历文件、证书材料 |
| Sui 对象模型 | 链上记录学历凭证、工作经历验证、技能认证 |
| Seal 加密 | 加密敏感职业信息，用户控制披露范围 |
| zkLogin | 支持 OAuth 登录关联职业身份 |

## 实现方案

### Agent / AI 部分

- 数据整理 Agent 从用户上传的材料中提取结构化职业信息
- 简历生成 Agent 分析目标岗位 JD，匹配相关经历并生成定制化简历
- 亮点提炼 Agent 从项目经历中提炼关键成就和量化指标
- 技能评估 Agent 根据代码提交、项目经历评估技能水平
- 差异分析 Agent 对比用户职业数据与目标岗位要求，识别技能差距

### Walrus 集成

- 职业数据以 `resume/{user_id}/profile.json.enc` 加密存储
- 工作经历以 `resume/{user_id}/experiences/{exp_id}.json.enc` 存储单条记录
- 简历文件以 `resume/{user_id}/resumes/{resume_id}.pdf` 存储生成结果
- 证书材料以 `resume/{user_id}/certificates/{cert_id}.json.enc` 存储
- 验证凭证以 Sui 链上对象存储，记录各经历的验证状态和验证方签名

### 前端 / 后端

- 前端：React 简历平台，支持数据管理、简历编辑、一键生成
- 后端：Python AI 服务 + Move 验证合约
- 关键页面：职业档案页、简历生成器页、岗位匹配页、验证面板页

## 技术架构

用户上传职业材料 → 数据整理 Agent 提取结构化信息 → Seal 加密后写入 Walrus → 用户选择目标岗位 → 简历生成 Agent 分析 JD → 亮点提炼 Agent 匹配相关经历 → 差异分析 Agent 生成改进建议 → 定制简历输出到 Walrus → 雇主请求验证 → 链上查询验证凭证 → 确认经历真实性 → 用户选择性披露详细信息。
