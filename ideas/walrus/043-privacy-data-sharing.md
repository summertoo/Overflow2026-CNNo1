# Privacy Data Sharing / 隐私保护的数据共享

> 使用 Seal 加密的 Walrus 存储，多方可共享数据但各自只能看到授权部分，实现可控数据协作

## 解决什么问题

多方数据协作中数据隐私与共享需求存在根本矛盾：企业需要共享数据进行联合分析，但又不能暴露原始敏感数据。传统方案依赖可信第三方，存在数据泄露风险。本方案利用 Walrus 存储加密数据，Seal 实现细粒度访问控制，不同参与方只能解密被授权的数据子集，在保护隐私的前提下实现多方数据协作。

## 核心功能

- 分级加密共享：数据提供者上传加密数据到 Walrus，按粒度为不同接收方设置解密权限
- 安全计算代理：Agent 在加密数据上执行分析任务，仅返回聚合结果而非原始数据
- 权限动态管理：数据所有者可随时更新授权策略，吊销访问权限实时生效

## 使用的 Walrus / Sui 能力

| 能力 | 用途 |
|------|------|
| Walrus Blob 存储 | 存储多方共享的加密数据集 |
| Sui 对象模型 | 链上记录数据授权策略、参与方列表、访问记录 |
| Seal 加密 | 按字段/按行级别的细粒度加密和访问控制 |
| Sui 权限模型 | 管理数据访问授权和吊销 |

## 实现方案

### Agent / AI 部分

- 加密 Agent 根据授权策略对不同数据字段使用 Seal 分级加密
- 访问控制 Agent 在每次数据请求时验证请求方的链上授权状态
- 安全计算 Agent 在授权范围内执行数据分析，确保结果不泄露原始数据
- 审计 Agent 记录所有数据访问行为，生成合规报告
- 隐私检测 Agent 在返回结果前检查是否存在隐私泄露风险

### Walrus 集成

- 加密数据以 `shared/{dataset_id}/data.enc` 存储在 Walrus
- 数据模式以 `shared/{dataset_id}/schema.json` 存储明文描述
- 授权策略以 Sui 链上对象存储，定义每方可访问的字段范围
- 访问日志以 `shared/{dataset_id}/access_log.json.enc` 加密记录
- 计算结果以 `shared/{dataset_id}/results/{task_id}.json` 存储

### 前端 / 后端

- 前端：React 数据共享平台，支持数据上传、授权管理、分析请求
- 后端：Python 数据分析引擎 + Move 授权合约
- 关键页面：数据上传页、授权管理页、安全分析页、审计日志页

## 技术架构

数据提供者上传数据 → 加密 Agent 按 Seal 策略分级加密 → 加密数据写入 Walrus → 链上注册授权策略 → 数据消费方请求访问 → 访问控制 Agent 验证链上授权 → 授权通过后 Seal 解密对应字段 → 安全计算 Agent 执行分析 → 隐私检测 Agent 检查结果 → 返回聚合分析结果 → 审计 Agent 记录访问日志。
